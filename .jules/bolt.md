## 2024-05-15 - CSS Layout Thrashing in Mouse Following Elements
**Learning:** Frequent updates to `top` and `left` CSS properties in a `requestAnimationFrame` loop cause layout thrashing, as the browser has to recalculate the layout on the main thread for every frame. This can drop frame rates and make animations feel sluggish, especially on lower-end devices or when the page has complex layouts.
**Action:** For animations that track cursor movement, always use `transform: translate3d(x, y, 0)` combined with `will-change: transform`. This shifts the animation work to the GPU's compositor thread, completely avoiding layout and paint operations on the main thread. Center the element initially using static negative margins or negative `top`/`left` offsets matching half its dimensions so the `translate3d` values map 1:1 to the cursor coordinates.
